v scenariju predvidimo, da bomo imeli dostop do mysql strežnika z username in password in to tudi naredimo doma:
   - odpremo firewall za port 3306 (mysql) / če ga npr. xampp še ni odprl
   - se zlogiramo na mysql server. Za potrebe remote dostopa naredimo novega uporabnika in morebiti bazo
   
        baza : remoteDB    // za test narediš tole kar s phpmyadmin, 
		tabela : testtabla
		
		user+privilegiji:         ( user:  pepi / password: pepipepi )
		

         CREATE USER 'pepi'@'%' IDENTIFIED VIA mysql_native_password USING '***';
		 GRANT ALL PRIVILEGES ON *.* TO 'pepi'@'%' REQUIRE NONE WITH GRANT OPTION MAX_QUERIES_PER_HOUR 0 MAX_CONNECTIONS_PER_HOUR 0 MAX_UPDATES_PER_HOUR 0 MAX_USER_CONNECTIONS 0;
		 GRANT ALL PRIVILEGES ON `remotedb`.* TO 'pepi'@'%';
		 
		 #opomba: user mora biti specificiran kot pepi@% ker mu le tako dajete dostop od kjerkoli, sicer je namesti %i IP mašine s katere delate povezavo
		 #opomba: zgornje je pobrano kot export iz mojega mysql serverja, delamo pa to takole:
		 
		 CREATE USER 'pepi'@'%' IDENTIFIED BY 'pepipepi';
         GRANT ALL PRIVILEGES ON remotedb.* TO 'pepi'@'%' WITH GRANT OPTION 
		 
		 po obeh variantah izvedeš NUJNO še:
		 
		 
		 flush privileges;
   
        sicer se lahko zgodi, da user ne bo uproaben do naslednjega reboota mysql serverja

     /usr/bin/mysql -u pepi -h 192.168.1.75 -p"pepipepi" remoteDB

	 # raje uporabite --password=pepipepi ker narekovajev ni treba in je bolj ugodna sintaksa za skripto )

  recimo, da kreiramo testno. Naslednjo vsebino shranimo v datoteko crTable.sql :
  
use remoteDB; 
CREATE TABLE `testtable` (
  `id` int(11) NOT NULL,,
  `timestamp` timestamp ,
  `vrednost` decimal(10,0) NOT NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8 COLLATE=utf8_slovenian_ci;

ALTER TABLE `testtable`
  ADD PRIMARY KEY (`id`);

ALTER TABLE `testtable`
  MODIFY `id` int(11) NOT NULL AUTO_INCREMENT;
COMMIT;


nato izvedemo :

     /usr/bin/mysql -u pepi -h 192.168.1.75 --password=pepipepi remoteDB < crTable.sql
